Corrupt Witness

Create binary files w/ build in 'grepouts' to remove dirty words from ps, netstat, ss, and strings output

Table of Contents

Usage

Information

Work With: Linux
Requires:

Corrupt Witness

Corrupt Witness is just a basic grep out. You give it some strings, it gives you replacement binaries for ps, netstat, ss, and strings that will run a grep -v with them, removing your strings from the output.


Usage

/dragonfire/bin/corrupt_witness/corrupt_witness.py
This will start corrupt_witness, select the binaries you want to build (normally all of them) and then choose 'Finished.'

Next, enter the strings you with to remove from the output. For example, if you want to remove 'bash' and references to port 22 in netstat you would enter 'bash,:22' You should attempt to be as specific as possible when writing your string matches. For ports I use :22 instead of just 22 otherwise everything with '22' in it would be removed, which would likely also include many valid pids in the process list, etc.

Once finished, the files will be written to /dragonfire/bin/corrupt_witness/output. This folder contains the modified binaries as well as install directions. The install directions include the 'mv' commands you'll need to 'back-up' the legitimate on-box binaries. The modified binaries call these real binaries, so they must be placed exactly where the install directions indicate.

After renaming the on-box binaries, place the modified binaries into the expected location. /bin/ps, /bin/netstat, /bin/ss, and /usr/bin/strings Of course these are the expected locations, you may have to adjust if binaries are in a non-standard location.

Now the binaries should be working. Run a couple tests to confirm your output is stripped. You can directly call the 'backed-up' binaries if you want a comparison.

To undo, move the 'backed-up' binaries over the modified binaries using the commands in the install text.
